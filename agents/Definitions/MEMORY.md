# MEMORY.md Standard

## Purpose

`MEMORY.md` stores curated, durable lessons and patterns that should influence future behavior.

It is not a transcript, scratchpad, task list, or place for everything the agent ever learned.

## Target size

- Ideal: 500–2,000 words for bootstrap memory.
- Hard ceiling: 3,000 words.
- Move older or detailed memory to dated notes, ENGRAM, or project docs.

## Required sections

1. `# MEMORY.md — Lessons & Patterns`
2. Short scope note: what belongs here and what does not.
3. Entries grouped by theme or date.
4. Each entry includes:
   - `## <Lesson Title> (<YYYY-MM-DD>)`
   - Context in 1–2 sentences.
   - Rule or future behavior in 1–3 bullets.

## Optimization rules

- Only hard-won lessons belong here.
- Keep entries actionable.
- Merge duplicate lessons.
- Promote cross-agent lessons to shared `PATTERNS.md`.
- Archive stale detail into ENGRAM or dated memory.

## Anti-patterns

- Raw chat logs.
- Daily journaling.
- Active task status; use CORTEX.
- Preferences better stored in `USER.md`.
- Tool commands better stored in `TOOLS.md`.

## Evaluation checklist

- Every entry changes future behavior.
- No duplicate lessons.
- No secrets or raw private data.
- No stale active task state.
- File stays small enough for bootstrap.
