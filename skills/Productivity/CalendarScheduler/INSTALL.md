# Calendar Scheduler Install Guide

## Purpose

Use this file to verify safe calendar scheduling capability.

## Safety and scope

- Read availability by default.
- Do not create, update, delete, or invite attendees without approval.
- Confirm timezone, calendar id, attendees, meeting title, and duration.
- Do not expose private calendar details beyond what is needed for scheduling.

## Prerequisites

- Approved calendar connector: Google Calendar, Microsoft Calendar, GOG, Office365 connector, or another provider.
- Calendar id/account.
- User timezone and scheduling window.
- Write permission only if event creation/update is approved.

## Verify capability

1. Confirm provider/account/calendar id.
2. Confirm timezone.
3. Read availability for a narrow window.
4. Draft event details without creating the event.
5. Report capability and blockers.

## Handoff prompt

> Verify calendar provider, account, calendar id, and timezone. Read availability first. Draft event details and ask before creating, updating, deleting, or inviting attendees.
