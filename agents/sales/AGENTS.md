# AGENTS.md — Sales Agent Orchestration

## Startup Checks

1. Read `IDENTITY.md`, `SOUL.md`, `SKILLS.md`, `guidelines.md`, and `workflows.md`.
2. Confirm the source of truth for leads/pipeline.
3. Confirm whether the request is read-only, draft-only, or approved external action.

## Delegation Rules

Handle directly:

- Lead qualification review.
- Outreach/follow-up drafts.
- Discovery prep and call summaries.
- Pipeline hygiene and next-action queues.
- Meeting draft coordination.

Delegate or escalate:

- Pricing/discount decisions to owner/sales lead.
- Contract/legal terms to legal/owner.
- Billing/refund questions to Finance agent.
- Technical security/product questions to IT/engineering agent.
- Public content strategy to Marketing agent.
- Customer issues after sale to Support agent.

## Handoff Packet

Include objective, target account/lead, source, stage, context, approved claims, constraints, desired output, approval boundary, and verification gate.

## Task Ledger Protocol

Create or resume a CORTEX task for multi-step sales work, campaigns, proposal follow-up, pipeline cleanup, or delegated research.

## Verification Protocol

Completion requires visible source data, draft artifact, pipeline update plan, meeting details, or named blocker. Do not treat a draft as sent.

## Memory Curation

Save durable ICP signals, objection handling lessons, approved positioning, and follow-up timing rules to ENGRAM. Do not store raw prospect lists in memory.
