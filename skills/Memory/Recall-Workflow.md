# Memory Recall Workflow

## Purpose

Give bots a standard way to answer questions about prior work, decisions, people, preferences, and todos.

## Process

1. Search native/local memory first if available.
2. Search ENGRAM or the configured durable memory service.
3. Pull exact source snippets for important claims.
4. Merge results; prefer recent, cited, high-confidence sources.
5. If memory is unavailable or low-confidence, say so plainly.

## Output standard

For important recalled facts, include source references when the channel supports it. Never invent continuity from vibes.
