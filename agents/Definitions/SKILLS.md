# SKILLS.md Standard

## Purpose

`SKILLS.md` defines the approved skill set for a role-based agent and when to read each skill.

It should route the agent to golden skill instructions instead of copying full runbooks into the agent folder.

## Target size

- Ideal: 300–900 words.
- Hard ceiling: 1,500 words.

## Required sections

1. `# SKILLS.md — <Role> Agent`
2. `## Core Skills`
3. `## Conditional Skills`
4. `## Skill Loading Rules`
5. `## Missing Skill Protocol`
6. `## What Does Not Belong Here`

## Optimization rules

- Link to skill README pages.
- Describe when to use the skill in one sentence.
- Keep role baseline lean; use conditional skills for infrequent work.
- Avoid copying skill content.

## Anti-patterns

- Full install instructions.
- Tool commands.
- Unapproved or unverified skill links.
- Skills included only because they exist.

## Evaluation checklist

- Every skill supports a role responsibility.
- Conditional skills are clearly labeled.
- Links are valid.
- Missing skill behavior is defined.
