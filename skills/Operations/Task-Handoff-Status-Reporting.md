# Task Handoff and Status Reporting

## Purpose

Make work transferable between bots, humans, and future sessions.

## Handoff packet

Include:

- Objective.
- Current state.
- Files/repos/services touched.
- Commands already run.
- Verification already performed.
- Known blockers and next action.

## Status style

Use concise bullets. Distinguish facts from assumptions. Mark blockers explicitly as `[blocked]`.
