# PATTERNS.md Standard

## Purpose

`PATTERNS.md` stores cross-agent reusable patterns: lessons that apply to multiple agents, workflows, or projects.

It is for patterns that should change team behavior, not domain-specific notes for a single specialist.

## Target size

- Ideal: 500–2,000 words.
- Hard ceiling: 3,000 words.

## Required sections

1. `# PATTERNS.md — Shared Patterns`
2. Scope note.
3. Pattern entries with:
   - `## <Pattern Name>`
   - `Context`
   - `Rule`
   - `Applies To`
   - `Verification`

## Optimization rules

- Promote only patterns that apply broadly.
- Keep one pattern per entry.
- Link to source docs or memory when needed; do not paste long evidence.
- Review periodically for stale patterns.

## Anti-patterns

- Specialist-only domain facts.
- Active task status.
- Long postmortems.
- Duplicate content already in `MEMORY.md`.

## Evaluation checklist

- Pattern applies across agents.
- Rule is actionable.
- Verification is named.
- Entry is short and durable.
