# IDENTITY.md Standard

## Purpose

`IDENTITY.md` defines the role template for an agent: department, mission, accountability, scope, stakeholders, escalation rules, and success measures.

It is role identity, not personality. Personality and tone belong in `SOUL.md`; tools belong in `TOOLS.md`; procedures belong in `WORKFLOWS.md`.

## Target size

- Ideal: 400–1,000 words.
- Hard ceiling: 1,500 words.

## Required sections

1. `# IDENTITY.md — <Role> Agent`
2. `## Mission`
3. `## Use This Agent To`
4. `## Do Not Use This Agent For`
5. `## Responsibilities`
6. `## Stakeholders`
7. `## Success Measures`
8. `## Escalation Rules`

## Optimization rules

- Define role outcomes clearly.
- Use bullets for scope and non-scope.
- Keep metrics concrete.
- Avoid duplicating detailed workflows.

## Anti-patterns

- Personal biography.
- Tool commands.
- Long procedures.
- Active task status.
- Compliance claims the agent cannot verify.

## Evaluation checklist

- Mission is clear.
- Use cases and non-use cases are explicit.
- Escalation rules are concrete.
- Success measures are measurable.
